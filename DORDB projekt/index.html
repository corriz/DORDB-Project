2. Slovní popis dodatečných integritních omezení

  1. Datum vypůjčení automobilu musí být nižší než datum vrácení automobilu.
  2. Při vytvařeni vypujčky musí být řidič i vozidlo ze stejné pobočky.

3. Slovní návrh API rozhraní „business logiky“ pro dva procesy

  1. Přidání vypůjčení
    Na základě osobního čísla a spz vozidla je založena výpujčka. Při zadládání se otestuje zda je možné vozidlo zapůjčit. Kontroluje se u již existujících záznamů datum_vypujceni a datum_vraceni, tak by se datumy v žádném případě nepřekrývaly.
  2. Přidání jízdy
    Na základě osobního čísla řidiče, spz vozidla je založena jízda, která odpovídá určitému datumu a také se zaeviduje start a cíl jízdy. Kontroluje se především zda exustuje nějaká výpujčka ke zvolenému datu a řidiči s vozidlem.

4. Pět slovně formulovaných dotazů nad schématem

  1. Vypiše všechny řidiče, kteří najezdili více jak 35000km
  2. Vypíše automobil a model (podle ujetých kilometru) a jejich model.
  3. Vypíše všechny řidiče, kteří za poslední rok udělali víc jak 20 jízd
  4. Vypíše (všechny) datum vypujčení, model, sqz a jméno řidiče, podle vzdálenosti, ktrou ujeli na jedno vypůjčení.
  5. Vypíše všechny řidiče, kterí řídili vozidlo, model MAN
